## Anti-pattern: files > 1500 lines without impl-level boundaries

Captured as a knowledge-base entry alongside this audit. Symptoms:

- Reviewers need a full-file search to find a handler by path
- Multiple `impl` blocks or `mod` sections exceed 400 lines each
- Test module at the bottom is itself 500+ lines and exercises concerns
  from >3 of the parent `impl` blocks

Mitigation pattern:

1. Identify natural seams (`impl` boundaries, request/response types vs.
   handlers vs. middleware).
2. Split the file at the seam with the fewest cross-references first.
3. Keep the original file as a `mod.rs` that re-exports the public
   surface so external callers need no changes.
4. Extract tests to `<name>_tests.rs` at the same level.

## CI guardrail

A lint step in `.github/workflows/rust-lint.yml` that fails any PR
introducing a new `*.rs` file over 1,500 lines is an appropriate
follow-up (tracked separately; not part of this audit's deliverable so
it doesn't gate archiving).
