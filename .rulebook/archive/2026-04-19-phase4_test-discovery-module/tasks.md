## 2. Test files

- [x] 2.1 Wrote `tests/discovery/basics.rs` — 3 tests: `Discovery::discover` against an empty store, against a one-collection store (verifies `collections_searched` + `queries_generated` metrics populate), and the default exclude list (`*-test`, `*-backup`) correctly drops blacklisted collections.
- [x] 2.2 **Re-scoped from `edge_cases.rs` (symlinks / unreadable files / empty dirs) to `filter_score.rs`**: `src/discovery/` is the **search-result orchestration pipeline** (filter → score → expand → broad → focus → compress → readme → plan → render), not a filesystem walker. Verified via `grep -rn "fn read_dir|symlink|workspace\.yml|canonicalize" src/discovery/` — zero matches. Symlink/unreadable-file concerns live in `src/file_loader/` and `src/file_watcher/`; testing them here would have tested the wrong code. The new `filter_score.rs` provides 8 tests covering `filter_collections` (no-input / include-only / exclude-only / no-match / empty input) and `score_collections` (descending-score ordering, empty-terms, empty-collections).
- [x] 2.3 **Re-scoped from `path_traversal.rs`**: there is no path-traversal surface in `src/discovery/` (verified by the same grep as 2.2). The `phase2_sanitize-discovery-paths` task mentioned in the original item scope guards path input in the FILE LOADER module, not here. Path-traversal testing belongs in that task's test slot. Removing this item from the discovery suite is the correct action, not a deferral — there is nothing in this module to test against path traversal.
- [x] 2.4 **Re-scoped from `manifest_parse.rs` (YAML + proptest)** for the same reason: `src/discovery/` does not parse any YAML manifest. `workspace.yml` is parsed in `src/config/vectorizer.rs::VectorizerConfig` and loaded in `src/server/core/workspace_loader.rs`; testing those files here would be miscategorised. Replaced by `expand.rs` (6 tests on `expand_queries_baseline`: each toggle in isolation, `max_expansions` truncation, `max_expansions = 0`, stopword stripping, empty query) and `compress.rs` (5 tests on `compress_evidence`: empty/single/many chunks, per-doc cap, global bullet cap, descending-score ordering).
- [x] 2.5 Wrote `tests/discovery/concurrent.rs` — 2 tests. `concurrent_discover_calls_are_safe_and_consistent` fires 16 parallel `discover` calls against the same `Discovery` instance and asserts every call reports the same `collections_searched` count (deterministic) and no panic/deadlock. `concurrent_discover_with_distinct_queries_does_not_interfere` cycles 3 different queries through 24 concurrent tasks and asserts every one completes without error. The re-scope swapped "re-index + watcher events" (file-watcher concern, belongs elsewhere) for pure read-side pipeline concurrency which is the only concurrency exposed by this module.
